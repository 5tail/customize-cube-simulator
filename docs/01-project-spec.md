# 01 專案規格：小丸號客製方塊模擬器（客製化圖像印製魔術方塊模擬網站）

> 這份是「我們在做什麼」的唯一定義。功能要不要做、先做哪個，以這份為準。修改本檔需使用者同意（見 docs/06）。

## 一句話
讓客人在網頁上把自己的圖片貼到 3D 魔術方塊上預覽，滿意後送出訂製需求；店家收到通知、建檔、報價；客人可產生臨時分享連結給別人看。

## 使用者角色
- **客人**：不註冊也能用模擬器；送件時留聯絡方式。
- **店家（= 專案擁有者）**：收通知、看訂單、報價。只有一人，不需要多店家系統。
- **被分享者**：點臨時連結看方塊預覽，無其他權限。

## 功能分期（嚴格照順序做，每期完成並上線才做下一期）

### Phase 1 — 模擬器（純前端，可先不碰後端）
- 3D 魔術方塊（3x3），可拖曳旋轉觀看。
- 上傳圖片（jpg/png，單檔 ≤ 8MB；原 5MB，使用者 2026-07-05 調高），選擇貼到哪一面；六面可各自不同圖。
- 圖片可在面上縮放/位移做簡單裁切。
- 圖片可在面上**旋轉**：順時針/逆時針各最多 180°（使用者 2026-07-05 新增，用途：六面拼接圖片的精確對位）。
- 「打亂／復原」按鈕（使用者 2026-07-05 新增）：動畫式單層旋轉模擬真實打亂，**純展示用**，不影響送件內容；復原＝回到整齊狀態。
- **手動撥層**（使用者 2026-07-05 追加）：按在方塊上拖曳＝撥動該層（放開自動吸到 90 度倍數）；按空白處拖曳＝旋轉視角。模擬真實玩轉方塊的操作，手機與電腦通用。
- 完全在瀏覽器端運作，不上傳任何東西到伺服器。
- 驗收：手機和電腦瀏覽器都能操作。

### Phase 2 — 一鍵截圖＋費用試算（純前端收尾，2026-07-05 使用者改定此路線）
- **一鍵截圖**：一顆按鈕把目前 3D 模擬畫面存成圖片下載（客人再自行把截圖＋六張原圖傳給店家，用 LINE/email 等既有管道，**不做站內送件**）。
- **訂做費用試算**：依客人目前的設定即時顯示試算價格。**計價參數放 repo 裡的獨立設定檔**，店家用 GitHub 網頁直接改數字、commit 後自動重新部署生效，不需要改程式。計價公式的參數由店家提供後定案。
- 驗收：手機和電腦都能下載截圖；改一次設定檔參數，網站價格跟著變。

## 明確不做（沒有使用者書面同意前，任何 session 不得自行加入）
- 線上金流／付款
- 客人會員系統（客人不需帳號）
- 多語系、多店家、購物車
- 2x2 / 4x4 以外尺寸的方塊（先把 3x3 做好）
- 任何 AI 圖像生成功能

## 上線防範機制備忘（Phase 2 起適用）
- 所有密鑰（Supabase service key、Resend key）只放 GitHub Secrets 與 Supabase 環境變數，**絕不寫進程式碼或 commit**。
- 上傳檔案驗 MIME type 與副檔名，重新命名為隨機檔名再存。
- Supabase 資料表全部開 Row Level Security（RLS），預設拒絕，逐條開放。
- 送件頁加使用條款一句話：「上傳圖片之版權責任由上傳者自負」。
