## L-001 店家可編輯的設定檔，測試不准寫死具體數值（2026-07-06）
情境：使用者自行到 GitHub 網頁改 `pricing.json` 的價格，main 的 CI 測試失敗、build/deploy 被跳過，網站停留在改價前的舊版本，使用者以為改價沒生效。
根因：`quote.test.js` 把使用者最初給的具體價格數字寫死當測試期望值（例如 `expect(...).toBe(420)`），沒有把「業務數值」和「計算邏輯」分開測。
規則：任何設計成給使用者自行編輯的設定檔（pricing.json 之類），測試只驗證**結構與邏輯**（用檔案內建的假資料跑計算函式；對真實檔案只檢查型別、範圍、遞增遞減等結構規則），絕對不把使用者當下填的具體數字寫進 `toBe(...)` 斷言。
